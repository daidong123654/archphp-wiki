命名规范

  1. 新建一个Action类
    • Action类负责项目的逻辑控制
    • cd Action #转到项目的Action目录下
    • touch test.class.php #比如我们建立一个名字是test的Action
    • vim test.class.php #编辑这个文件如下 
      	<?php
	class TestAction
	{		
	}
    • 这样就建立了一个Action类
    • 或者可以这么定义(如果需要的话) 
      	<?php
	class TestAction extends Action
	{
	}
      所以Action类的命名规范是:类文件名字全小写类似 test.class.php;类名字双峰式类似: TestAction
  2. 新建一个Mod类
    • Mod类负责项目的模型建设(怎么说呢,如果你熟悉MVC模式就不用去了解了).Mod即Model之意
    • 完全类似于Action的命名规则:类文件名字全小写类似 test.class.php;类名字双峰式类似: TsetAction
    • 另外,Mod的类一般需要继承框架的类Mod,所以需要像下面这样定义一个Mod类 
      	<?php
	class TestMod extends Mod
	{
	}
  3. 新建一个Addon类
    • 完全与前两者一样.在项目中Addon类是出于需求而使用的第三方类库(比如PHPmailer,PHP短信接口,PHP缩略图类等).
    • 框架本身不会去提供这些类
    • 类文件名字全小写类似: test.class.php;类名字双峰式类似: TestAddon
    • 另外:Addon类定义时不需要继承框架的类
