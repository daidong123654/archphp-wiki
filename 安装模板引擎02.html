安装模板引擎

3.简单地安装Tpl

  • 说实话这个模板引擎的名字我还真不知道,在OSC上看到的,真是个好东西
  • 下载:https://github.com/hit9/hit9mvc/blob/master/Tpl/tpl.class.php

安装到ArchPHP中

  • 1.安装为Action的属性: 把文件tpl.class.php放到目录Core/Action/tpl/下,并在Core/Action/action.config.php的数组中加入'tpl'即可 .然后在Core/Config/config.php中加入配置
    • 如何使用:
    • 详细的模板语法在tpl.class.php文件中已有.
    • 这里只写一个测试安装成功的例子
    • 首先在Core/Config/config.php中加入的配置: 
      		/*
		 *Tpl配置
		 */
		define('TPL_DIR','Tpl');//模板存放目录,相对于入口文件
		define('TPL_C_DIR','Tpl_c');//模板缓存目录,相对于入口文件
		define('TPL_REAL',true);//实时编译,开发完成后关掉
    • 然后写一个测试: 
      		<?php
		class IndexAction extends Action
		{
			public function index()
			{
				$this->tpl->assign('var','hello');
				$this->tpl->display('sample');
			}
		}
      相应的Tpl/文件夹下的sample.html模板是: 
      		<html>
		<head>
			<meta http-equiv="content-type" content="text/html; charset=utf-8">
			<title>Sample</title>
		</head>
		<body>
			{$var}
		</body>
		</html>
  • 2.安装为Action的父类
  • 把模板放到Core下随便一个目录 比如Core/Tpl 然后在Core/core.php中引入这个类:
    • require_once(CORE_PATH.'Tpl/tpl.class.php');//注意此句需要在Core/Action/action.class.php引入之前
  • 然后修改Core/Action/action.class.php:(继承Tpl类)
    • class Action extends Tpl
  • 配置Tpl类,同方法一:(Core/Config/config.php中加入以下配置) 
    		/*
		 *Tpl配置
		 */
		define('TPL_DIR','Tpl');//模板存放目录,相对于入口文件
		define('TPL_C_DIR','Tpl_c');//模板缓存目录,相对于入口文件
		define('TPL_REAL',true);//实时编译,开发完成后关掉
  • 然后按照开始测试! 
    		<?php
		class IndexAction extends Action
		{
			public function index()
			{
				$this->assign('var','hello');
				$this->display('sample');
			}
		}
    相应的模板文件sample.html(Tpl目录下) 
    		<html>
		<head>
			<meta http-equiv="content-type" content="text/html; charset=utf-8">
			<title>Sample</title>
		</head>
		<body>
			{$var}
		</body>
		</html>
